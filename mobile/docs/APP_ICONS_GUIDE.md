# App Icons and Labels Management Guide

This guide explains how to manage app icons and labels for both Client and Merchant apps.

## App Labels

App labels are configured in `android/app/build.gradle.kts` for each flavor:

- **Client App**: "Wedy"
- **Merchant App**: "Wedy Business"

Labels automatically include environment suffixes (e.g., "Wedy (dev)") based on `AppConfig`.

To change labels, edit the `resValue("string", "app_name", "...")` in `build.gradle.kts`.

## App Icons

### Setup

1. **Prepare Icon Images**
   - Create square icon images (1024x1024px recommended)
   - Place them in `assets/icons/`:
     - `client_icon.png` - Client app icon
     - `merchant_icon.png` - Merchant app icon

2. **Generate Icons**

   **For Client App:**
   ```bash
   flutter pub run flutter_launcher_icons -f flutter_launcher_icons_client.yaml
   ```
   This uses the configuration in `flutter_launcher_icons_client.yaml`.

   **For Merchant App:**
   ```bash
   flutter pub run flutter_launcher_icons -f flutter_launcher_icons_merchant.yaml
   ```
   This uses the configuration in `flutter_launcher_icons_merchant.yaml`.

### Manual Icon Setup for Flavors

Since `flutter_launcher_icons` doesn't directly support Android flavors, you'll need to manually copy icons:

1. **After generating client icons:**
   ```bash
   # Icons are generated in android/app/src/main/res/mipmap-*/
   ```

2. **For merchant app, copy icons to flavor-specific folders:**
   ```bash
   # Create merchant flavor resource directories
   mkdir -p android/app/src/merchant/res/mipmap-hdpi
   mkdir -p android/app/src/merchant/res/mipmap-mdpi
   mkdir -p android/app/src/merchant/res/mipmap-xhdpi
   mkdir -p android/app/src/merchant/res/mipmap-xxhdpi
   mkdir -p android/app/src/merchant/res/mipmap-xxxhdpi
   
   # Copy merchant icons (after generating with merchant config)
   # You'll need to manually copy from the generated icons
   ```

### Alternative: Use Adaptive Icons

For better Android support, use adaptive icons:

1. **Create adaptive icon foreground/background:**
   - Foreground: Your app icon (transparent background)
   - Background: Solid color or gradient

2. **Update `flutter_launcher_icons_client.yaml`:**
   ```yaml
   flutter_launcher_icons:
     android_adaptive_icon:
       foreground_image: "assets/icons/client_icon.png"
       background_color: "#5A8EF4"  # Your primary color
   ```

3. **For merchant, update `flutter_launcher_icons_merchant.yaml`:**
   ```yaml
   android_adaptive_icon:
     foreground_image: "assets/icons/merchant_icon.png"
     background_color: "#5A8EF4"
   ```

### iOS Icons

iOS icons are automatically generated by `flutter_launcher_icons`. The configuration in `flutter_launcher_icons_client.yaml` handles iOS.

## Quick Reference

### Change App Label

Edit `android/app/build.gradle.kts`:
```kotlin
create("client") {
    resValue("string", "app_name", "Your New Label")
}
```

### Regenerate Icons

```bash
# Client app
flutter pub run flutter_launcher_icons -f flutter_launcher_icons_client.yaml

# Merchant app  
flutter pub run flutter_launcher_icons -f flutter_launcher_icons_merchant.yaml
```

### Icon Requirements

- **Android**: 
  - Standard: 48dp (mdpi), 72dp (hdpi), 96dp (xhdpi), 144dp (xxhdpi), 192dp (xxxhdpi)
  - Adaptive: 108dp foreground, 432dp background
- **iOS**: 
  - App Icon: 1024x1024px
  - Various sizes auto-generated

## Notes

- Icons are cached by the OS, so changes may not appear immediately
- Uninstall and reinstall the app to see new icons
- For production builds, ensure icons are properly signed
- Adaptive icons provide better visual consistency on Android
